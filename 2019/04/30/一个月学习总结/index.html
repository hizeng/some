学习总结

Posted on 2019-04-30 | In skill

学习总结

一.学习收获

  • 打字速度增加到一倍（200／min)

  • 完成了4个任务（分开总结）

二.学习分析

自身技术条件与学习结果对比分析

  • html css 没有完全掌握的情况下 对于完成任务的逻辑控制部分断层

    前端页面 是数据渲染的产物 而 数据渲染的方式 则取决于 开发所运用的对象（如 html js控制Dom)

    样式的变化，数据的插入载体都是离不开页面的结构 （对于样式的 css语法 模式（less等）

    页面的交互 数据的存储 便衍生出（路由 服务器 数据库 等后端技术知识）

    项目的最终上线部署 （这里就开始要运用 打包构建工具的使用）

对于任务的逻辑 没理解透切 就迫于任务的完成时间 凑合完成 导致后续任务受前面任务涉及的知识 没完全掌握 和理解 而越做越慢

  • 基础差 导致学习的难度加大 消化的知识点 多而杂 是在做的过程当中 形成对于 web开发的理解
    我本来是学习web出身，但是由于 学习的路径 和 持续性 导致 一个月前学习的时候 基础基本接近于0
    但是还是依靠朱华宇 被硬扯上来 目前对于整个web 开发 有了一个较为全面的认识 和 认识自身技术的缺失。
    完成任务 需要对于 完全掌握有差距 也需要回头多加练习 不能再以基础差为学习效率差的理由

思考：

任务完成度 固然重要 但是 同时 要每天安排时间 去补足理论知识
学习目的是 ：如何在工作中开发，而不是如何完成这几个任务 必须学习的是 开发的思路 完成项目的各个关键步骤 知识点
解决问题的路径 开发的正确手势 简称程序开发的套路

再思考：

关于个人作息，记忆力是一个较为重要的点，时间规划的目的是 再时间段内 完成不了 就跳过 否则 其他任务规划 无法执行 这个规划没有任何意义了
适度放松 发觉自己 太死板 思维会出现僵化 他妈明明自己思维 本来是敏捷快速 为何 会僵化 目前分析的结果就是 他妈的生活和身边婆妈的人不断扰乱 导致自己出现 沉思走正事之神 容易殆尽自己的精力去做 根本以外的事

继续思考：

接触两年 入行一年 弯路 无数 为何？ 因为 自己从来没真正去思考过 编程的核心的什么？ 知识的核心？ 记忆要点知识？ 不断重复别人的理解 别人的理解之上 ，我写代码的目标有两个 第一 钱 第二 钱 程序知识太多 一定要专心自己的领域 因为 自己不是智力超群的人 只是 为了过生活 实现以后自己的其他想法 那么 就专心实现 月薪过万

围绕这个实现

有几点马上要做：

  1. 按任务完成10个要点（作为面试的依据）
  2. 打字速度破300 特别是 标点符号 ,./;’[]\=-()’<>?:”{}|+ “_（开发的气势）
  3. 样式和 html （开发的体现）
  4. js 的核心思想

接下来的几点：

  1. 工具的使用
